C. Canvas Painting

【题意】

序列上有 n(n≤109)n(n \le 10^9)n(n≤109) 个格子，编号为 1…n1\dots n1…n，起初编号为 iii 的格子上的颜色 colicol_icoli​ 为 iii。

有 mmm 次操作，操作顺序可以任意改变，每次操作给定一个区间 [li,ri][l_i,r_i][li​,ri​]，可以选择任意两个格子 u,vu,vu,v\in[li,ri][l_i,r_i][li​,ri​]，执行 colu=colvcol_u = col_vcolu​=colv​ 的操作。

你需要找到一个操作方案，使得最后剩下的颜色的种类最少，输出最少的颜色种类数。

【思考】

**Hint1：**每次操作至多能让颜色的数量减一，且只需当前区间并非全部一个颜色。

**Hint2：**把染色操作看成把一个格子的颜色挪到另一个格子上（更形象），跨着格子挪肯定不优秀，所以最后肯定颜色相同的是连续的一段。

考场上的其他思路：

  1. 区间操作顺序应该是怎么样的呢？猜测左端点排序。
  2. 还想了一些其他的性质，什么最左边的区间一定能减一啥的，都没什么用。
  3. 在Hint2加思路1的基础上，举了个大区间套小区间的反例，如下图

< img src="https://cdn.luogu.com.cn/upload/image_hosting/ri25sdyg.png" style="zoom:33%;" / >

​ 按左端点排序要先操作红色，但是存在红色操作完了绿色操作就操作不了的情况。

【题解】

记录一个 nw\text{nw}nw 表示当前最左的那段颜色延续到的最远距离。

  • 若存在操作区间 [li,ri][l_i,r_i][li​,ri​] 满足 li≤nw,ri>nwl_i \le \text{nw},r_i > \text{nw}li​≤nw,ri​>nw，把 rir_iri​ 放进堆里。每次取 rir_iri​ 最小的进行操作，操作完了之后 nw+1\text{nw} + 1nw+1，同时更新那个堆。
  • 若不存在区间满足，那把 nw\text{nw}nw 设置为当前未操作的区间中最小的 lil_ili​ 即可。

D. Min - Max Tree

【题意】

给你一棵树，每个点有点权 wiw_iwi​。

对于每个联通块，定义其权值为该联通块中的最大值减去最小值。

你可以断开任意条边，请让联通块权值和最大。

【思路】

考场上最开始想的是基于联通块合并的东西，现在有两个联通块最大最小值分别为 mn1,mn2,mx1,mx2mn_1,mn_2,mx_1,mx_2mn1​,mn2​,mx1​,mx2​，想了想什么情况下合并会优，什么情况下合并会劣，并基于此想了个贪心。仔细思考了一下，后效性太过于严重了，还举了反例。

然后想到一个联通块只有最大值和最小值是有用的，确定了这两个值，联通块只需要包含连接对应两个点的这条链。

从链延长或者断开的角度又想了想，发现选出链上的点权应该单调，不然在不单调处断开更优秀。

【题解】

现在的问题就是将树划分成若干条单调链，使得权值最大。

写了个树形dp，我把它叫树上插头dp哈哈。

fuf_ufu​：考虑了 uuu 子树，需要从 uuu 往外接一个升序链的情况下的最大权值。（ + 插头）

gug_ugu​：考虑了 uuu 子树，需要从 uuu 往外接一个降序链的情况下的最大权值。（ - 插头）

hu：h_u：hu​：考虑了 uuu 子树，不需要从 uuu 往外接链的情况下的最大权值。（无插头）

转移时，注意下合并的时候有 uuu 上面有两个插头的情况，特判一下。

I.Knapsack Problem

【题意】

给你一个无向图（n,mn,mn,m 都是 10510^5105），边有边权 w(u,v)w_{(u,v)}w(u,v)​，你有一些大小为 VVV 的背包。

从 SSS 出发，前往节点 TTT ，每次经过边 w(u,v)w_{(u,v)}w(u,v)​，背包物品体积加上 w(u,v)w_{(u,v)}w(u,v)​，若放不下就再来个新的背包。

问从分别 1…n1\dots n1…n 出发去到 TTT，各需要多少背包。

【思考】

从每个点出发，跑 (背包个数，体积)(背包个数，体积)(背包个数，体积) 的二元最短路正确性没问题。

最粗暴的想法是导致跑，仔细想想发现不对。

【题解】

还是考虑正着跑，发现每次换新背包的时候背包清空了，之前是怎么样的状态都不重要了，进一步思考发现花背包多的只会向着花背包少的跑。这样图就会根据花背包的多少分成一层一层的。

考虑倒着一层一层把这个图跑出来即可。

G. Sorting

【题意】

给出 m≤2×105m \le 2\times 10^5m≤2×105 对数对 (ai,bi)(a_i,b_i)(ai​,bi​) 其中 1≤ai<bi≤n≤2×1051\le a_i < b_i\le n \le 2\times 10^51≤ai​<bi​≤n≤2×105。运行如下进行排序，问能否把任意 nnn 的排列 ppp 都排好序。

【题解】

对于 n≤2×105n \le 2\times 10^5n≤2×105，我们认为 101810^{18}1018 轮交换足以将序列排序到最终状态。那么我们只需要考虑是否存在某一个排列 ppp 在当前状态下不能排好序。

将 ai→bia_i\rightarrow b_iai​→bi​ 画图首先我们可以发现，若存在 n→n−1→n−2→⋯→1n\rightarrow n-1 \rightarrow n - 2\rightarrow\cdots\rightarrow1n→n−1→n−2→⋯→1 的链，那么一定可以排。而对不存在的情况尝试构造反例后发现，若不存在则均能构造不能再排的序列。

不妨假设这条链上第一处断开的地方为 kkk 即存在 n→n−1→⋯→kn\rightarrow n - 1\rightarrow\cdots\rightarrow kn→n−1→⋯→k 而不存在 k→k−1k\rightarrow k - 1k→k−1 ，那么可以构造排列在 n,n−1,⋯k+1n,n - 1,\cdots k + 1n,n−1,⋯k+1 处对应放置 n,n−1,⋯ ,k+1n,n - 1,\cdots,k + 1n,n−1,⋯,k+1，而在 k−1k - 1k−1 处放置 kkk。这样一来 nnn 到 k+1k + 1k+1 的位置都已排好且一定不会再改变，因而 k−1k - 1k−1 处的 kkk 无法与后面值更小的交换，也无法与前面已经排好的交换，自然无法移动到 kkk 去。

J. Moving on the Plane

【题意】

平面上 n≤50n \le 50n≤50 个点位于 (xi,yi)(x_i,y_i)(xi​,yi​)，其中 0≤∣xi∣,∣yi∣≤1050\le |x_i|,|y_i| \le 10^50≤∣xi​∣,∣yi​∣≤105。经过 m≤105m \le 10^5m≤105 轮移动，每轮中每个点随机向上下左右四个方向移动。问有多少种方案使得移动结束后任意两点曼哈顿距离不超过 KKK，答案对 998244353998244353998244353 取模。

【题解】

首先考虑将曼哈顿距离转切比雪夫距离，x′=x+y,y′=x−yx'=x + y,y' = x - yx′=x+y,y′=x−y，那么限制变为 max⁡{Δx,Δy}≤K\max\{\Delta x,\Delta y\} \le Kmax{Δx,Δy}≤K。

同时移动从 (0,±1),(±1,0)(0,\pm1),(\pm1,0)(0,±1),(±1,0) 变成了 (±1,±1)(\pm1,\pm1)(±1,±1)。此时两轴上的运动独立，限制也独立了，于是可以分开考虑最后乘起来。

问题变成数轴上若干点随机移动。我们枚举最后所有点落在的区间 [L,L+k][L,L + k][L,L+k]，再枚举每个点用组合数计算落在该区间的方案数再相乘。为了去重，我们需要钦定最后存在点落在 LLL 上，这可以用不钦定时的 [L,L+k][L,L + k][L,L+k] 答案减去 [L+1,L+k][L+1,L + k][L+1,L+k] 答案得到。复杂度 O(nmk)O(nmk)O(nmk)。

M. Teleporter

【题意】

给定一个大小为 nnn 带边权的树，树上还有 mmm 条权值为零的额外边，求在经过不超过 0,1,⋯ ,n0,1,\cdots,n0,1,⋯,n 条额外边的情况下，从根到达每个点的最小路径边权和。

n≤5000,m≤10000n\le 5000,m\le 10000n≤5000,m≤10000

【题解】

暴力做法，以当前还能走多少条额外边分层，直接建分层图

如果建完分层图跑 dijstra\text{dijstra}dijstra，总的点数为 n2n^2n2，边数为 nm+n2nm + n^2nm+n2 复杂度是 O((n2+nm)log(n2))O((n^2 + nm)log(n^2))O((n2+nm)log(n2))。

注意到每一层只能在本层跑或者跑到下一层，考虑在每一层分开跑dijstra\text{dijstra}dijstra，复杂度 O(n(nlogn+m))O(n(nlogn + m))O(n(nlogn+m))，可以通过。
