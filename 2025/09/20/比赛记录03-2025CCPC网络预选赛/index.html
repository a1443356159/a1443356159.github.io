C. 造桥与砍树

【题意】

对于 n≤105n \le 10^5n≤105 的一张完全图，每个点有点权 0≤ti≤1090\le t_i \le 10^90≤ti​≤109，给出 1≤k≤1091\le k \le 10^91≤k≤109，则连接 u,vu,vu,v 的边的边权为 (tu+tv) mod k(t_u + t_v) \bmod k(tu​+tv​)modk，求最小生成树。

【题解】

考虑 boruvka，即每轮找到每个联通块向外的最小边后连接，每轮点数至少除以二，共 log⁡n\log nlogn 轮。对于每个点向外的最小边，考虑将所有点按点权排序，由于相加至多进位 111，我们发现每个点有一个对应的前缀都不会进位，剩下的后缀都会进位，且后缀的起始位置是一定的。那么二分找到这个位置记 stist_isti​。由于我们要找连通块以外的点，所以考虑每个点有颜色，于是我们要求的就是从某个位置 xxx 开始，颜色与 ccc 不同的第一个点。分别从 111 和 stist_isti​ 找一个取最优即可。转换一下也就是第一次出现第二个颜色的位置。

这个问题我们考虑每轮结束后，类似猫树地对这个序列，在线段树上每层的每个区间计算区间内的每个位置到右端点有多少颜色。每轮结束后计算对于每个位置，下一个相同颜色的位置在 nxtinxt_inxti​。那么就是计算 ∑xr[nxti>r]\sum\limits_{x}^r [nxt_i > r]x∑r​[nxti​>r]。每轮 nlog⁡nn\log nnlogn 地计算。然后，我们考虑如何从 xxx 开始找第一次出现两个颜色的位置，我们从线段树的叶子 xxx 处往上跳，一直跳到 xxx 到 rrr 的颜色数大于 111 的区间，那么此时答案在右区间，我们考虑若 midmidmid 和 mid+1mid + 1mid+1 颜色不同，那么 mid+1mid + 1mid+1 就是答案，否则我们相当于找 mid+1mid + 1mid+1 到 rrr 第一次出现两个颜色，也就是直接跳右儿子。然后我们就可以循环考虑跳做儿子还是返回 midmidmid 或是跳右儿子。这样就可以在 log⁡n\log nlogn 时间内找到答案，并且常数较小。

于是我们对每个点这样找到向外的边后，按边权排序依次连接即可。暴力地更新颜色并重新计算 nxtinxt_inxti​ 等即可。

总时间复杂度 O(nlog⁡2n)O(n\log^2 n)O(nlog2n)。

D. 通配符匹配

【题意】

给你一个至多含有10个通配符 ∗*∗ （可以匹配任意一个字符串或者空串） 的字符串 S(∣S∣≤5×105)S(|S| \le 5\times10^5)S(∣S∣≤5×105) ，给你一个文本串 T(∣T∣≤5×105)T(|T| \le 5\times 10^5)T(∣T∣≤5×105)，问 TTT 的子串中，有多少个可以与 SSS 匹配。

【思路】

因为只有10 个通配符，将每一段分开考虑。

记第 i−1i - 1i−1 的通配符与第 iii 个通配符之间的这部分字符串为 SiS_{i}Si​。

先考虑一个暴力的 dpdpdp。

设 f[i][l][j]f[i][l][j]f[i][l][j] 表示：子串 T[l…j]T[l\dots j]T[l…j] 是否能与 SSS 的以第 iii 个通配符结尾的前缀匹配。

转移的时候，枚举 Si+1S_{i + 1}Si+1​ 能匹配的位置记作 kkk：
f[i][l][j]→f[i+1][l][k],k≥j+∣Si∣ f[i][l][j]\rightarrow f[i+1][l][k],k \ge j + |S_i| f[i][l][j]→f[i+1][l][k],k≥j+∣Si​∣
【正解】

Hint1：在暴力dp的时候容易发现左端点这一维没有用。

Hint2：含多个通配符匹配的字符串匹配是可以贪心的，每次尽可能找前一个 SiS_iSi​ 匹配即可。

什么意思呢？

假如 SSS： a*ab*c，TTT：aababcc

a，ab，c都会尽可能靠前匹配。

a*ab<----->aab

a*ab*c < ---- - > aababc

为什么呢？a靠前匹配会让ab的"选择"多一些，进而让c的“选择”多一些。如果a，ab靠后匹配，那么匹配方案都能够通过调整*c的匹配来对应。

c靠前匹配，统计答案的时候在求个前缀和就行了。

设 f[i][r]f[i][r]f[i][r] 表示：当前考虑了以第 iii 的通配符结尾的 SSS 的前缀，贪心地匹配，右端点为 rrr 的子串对应的左端点 lll 的数量。

转移的时候，找到TTT 中， rrr 之后的第一个 Si+1S_{i + 1}Si+1​转移即可。

因为是贪心的匹配，每一个左端点 lll 只会对应一个右端点 rrr。

统计答案的时候，最后 S10S_{10}S10​（最后一个通配符后面那段）的匹配不是贪心地，所以需要求一个前缀和统计一下答案。

实际实现的时候，需要处理一些细节：

  • 第一个符号是字符/通配符
  • 最后一个符号是字符/通配符
  • 全局没有通配符
  • 全局只有通配符

【思路】

最后的结果一定是每门课程的能力值由某位老师确定。

设 dp[i][t][S]dp[i][t][S]dp[i][t][S] 考虑了前 iii 名老师，表示花费了 ttt 时间，其中 SSS 这些课程已经确定了能力值（即确定对应的老师）的情况下的最高收益。

dp[i+1][t+ti][S∪T]←dp[i][t][S]+val[i][T]dp[i+1][t + t_i][S\cup T]\leftarrow dp[i][t][S] + val[i][T] dp[i+1][t+ti​][S∪T]←dp[i][t][S]+val[i][T]

预处理一个 to[S]={{t,val}}to[S] = \{\{t,val\}\}to[S]={{t,val}} 表示确定的集合为 SSS 的时候，花费时间 ttt 能获得 valvalval 的价值。

【题解】

正解是一个经典技巧，把枚举子集优化掉了。

设 dp[i][t][S]dp[i][t][S]dp[i][t][S] 考虑了前 iii 名老师，表示花费了 ttt 时间，其中 SSS 这些课程已经确定了能力值（即确定对应的老师）的情况下的最高收益。

这是因为

  • 每门课程对转移的影响是独立的，没有什么一起选哪些课程可以获得更高分一说。
