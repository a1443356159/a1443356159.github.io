之前模拟赛做了不少状压dp的题，很多都是看似需要枚举子集，结果最后只需要一个一个元素转移。结合先前做状压dp的经验，重新思考了一下这道题该如何做。

【题意】

  • 矩阵第一行为 ，最后一行为 。

  • 四连通的 # 均为实心矩形。

  • 八连通的 # 全部连通。

【题解】

一个很好的构造是在第二/六排把第一/七排取补集即可把上面全连通。在次基础上只要上下找到一条路能连通就行。我们在第二排找到任意一个空，用第三排把这个空变成 # 并在第四排把两个 # 连起来即可。

需要特判整行全是 # 的情况。

E. 合成大西瓜

【题意】

给出一个 个点 条边的无向连通图，其中 是奇数。每个点有点权 。一次操作是选择 的一条路径，合并为一个点 ，将原本连接到 的边连到新点 ，新点点权变为 ，求只剩一个点时最大的点权。

【题解】

首先当 为奇数时一定存在方案将连通图变成一个点。我们考虑从高到低验证一个点是否有可能作为答案，由于答案是取 所以可以考虑合并操作中的 ，也就是说如果一个点从头到尾都能作为 去合并新点，那么这个点的点权就可以作为答案候选。那我们考虑把 放在路径中间，于是可以发现如果 两边不连通，可以把 的两边一一对应操作，如果某一侧点数较多可以先在那一侧内部操作若干次直到两侧点数相同；如果 两边连通，那就可以直接断成两边点数相同。

于是我们发现，一个点能作为答案候选，只需要度数大于 。考虑度数等于 的点，当我们需要考虑他作为 贡献答案时，比他大的点一定没有作为 贡献答案，否则就不用考虑他了。那么此时比他大的点度数也为 ，我们连接这个点 和一个比他大的度数为 的点 ，中间若干点一定可以通过若干次操作变成 ，此时这个点就可以作为 贡献答案。

最后的结论就是度数大于 的所有点和度数等于 的第二大的点可以作为答案。

H. str(list(s))

【题意】

在 python 中对一个字符串进行 str(list(s)) 操作会得到一个新的字符串。

例如 abc 变成 ['a', 'b', 'c']。再进行一次同样的操作就会变成 ['[', "'", 'a', "'", ',', ' ', "'", 'b', ...... 'c', "'"] 这样的很长的嵌套的串。

形式化的， 变成 的规则是：

  • 不是单引号时， 都是单引号，否则都是双引号。

  • 时， 是空格，否则是 [。

  • 时， 是逗号，否则是 ]。

给出一个由除空白字符以外的可见字符组成的 ，其中 ，设 表示迭代进行 次操作后得到的字符串。给出 ，求 $ 0 j < ps^k$ 中下标模 为 位置的字符 ascii 码之和，对 取模。

【题解】

如果这个规则不给那还需要观察，给出来之后就很 ez 了。直接设 分别表示第 轮，模 为 的位置上分别有多少个 [‘, ”] 和其他字符。然后每轮枚举每个位置，计算 的地方会新增多少个字符即可。复杂度 。

因为新增的字符只有这 种，原本的字符可以全部做完之后算位置，做的过程中作为 起个占位的作用。但需要注意原本的字符也可能有单引号，由于单引号会导致单双的改变，所以需要特殊处理。

然后由于最后一维有 维所以常数略大，取模和转移过程中需要一定的卡常。减少重复计算，减少取模次数。
