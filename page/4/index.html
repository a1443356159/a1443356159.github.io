【题解】

L.绝望线缕

【题意】
　　给定一个序列a和序列b，长度分别为n和m，每次可以选择一个(i,j)，使得 aia_iai​ 和 bjb_jbj​ 分别减一，注意a中每个位置只能选择一次，当某个值变为0时，把所有其他的数都减一，并且删去这个数，问是否存在一种操作方案可以把b序列全部删除。
　　n,m≤5×105,ai,bi≤109n,m\le 5\times 10^5,a_i,b_i \le 10^9n,m≤5×105,ai​,bi​≤109

【解法】
　　能够发现如果有解，那么一定存在一种合法的操作方案，使得在所有操作操作完之后，才发生爆炸，然后一连串带走所有的b。然后我们发现其实选择哪个 aia_iai​ 并无区别，所以我们直接统计能够操作 - 1的 aia_iai​ 的个数。所有非1的都可以贡献一次，所有的1一共可以贡献一次。
　　那么我们就需要拿这些操作次数去修剪b序列，让b序列可以引爆之后一次性全爆炸完。
　　先排序，从小到大考虑，记录一个当前的爆炸次数 tot，如果现在这个 bi≤totb_i \le totbi​≤tot，那么就直接让tot加一，不然就先使用 bi−totb_i - totbi​−tot 次操作，再让tot加一，记得每次更新tot之后还要看a序列里有没有可以爆炸的元素。如果某次操作使得操作次数变成了负数，那么就不存在方案，否则就一定存在。

D.套娃

【题意】
　　给定一排大小不同的套娃，每次可以将相邻两个可以套娃套在一起，问最多可以套几次，能够套在一起的要求是相邻两个中有一个套娃中的最小那个比另一个套娃中最大的那个还要大。
　　n≤105n \le 10^5n≤105
【解法】
　　这个题很容易考虑错，比如笛卡尔树，不能用笛卡尔树做的原因是，一个节点的左右子树是可以合并到一起的，哪怕两个子树不是完全的大小包含关系也是有可能的，所以难以用笛卡尔树解决。
　　寻找性质，发现如果一段区间可以合并到一起，那么他的任一一个子区间也可以合并到一起，这样我们就可以进行二分，如果说我们固定了左端点，那么可以通过二分得到最远的右端点，最后的答案就是 n - 段数。
　　判断一段是否可以合并到一起，可以通过贪心去做，从左到右，每次如果碰到可以紧密相连的段（所以需要离散化），就合并到一起，如果最后只剩一段那么就可以，否则一定不可以。证明略，自行感受一下即可。判断一次的复杂度是 O(lenlog⁡(len))O(len\log(len))O(lenlog(len))，len是判断的区间长度，瓶颈是离散化。
　　如果我们从头开始，每次通过二分找到右端点，那么复杂度是错误的，所以我们要先倍增到不合法的第一个位置，再去区间内二分。时间复杂度 O(nlog⁡2n)O(n\log ^2 n)O(nlog2n)。

B. 括号

【大致题意】
　　给你一个包含 "()" "[]" " < > " "{}" 四种括号的字符串。
　　再给你 mmm 个它的字串，问有多少对子串能拼成一个合法的括号序列。
　　n,m≤5e5n,m \le 5e5n,m≤5e5
【题解】
　　两个能拼成一个合法子串的括号序列，在分别完成自身内部的括号匹配之后一定是形如"(<{ < "和">} > )"的形式，即后一个串反向后与第一个串相同。这个是否相同用哈希判断即可。
　　我们分配求出拼在前面的那个串的哈希值和拼在后面的那个串的哈希值，容易发现，要算拼在后面的那个串的哈希值，只需要将原字符串反向，再将左右括号对换，就和求前面那个串的哈希值是一样的问题了。
　　我们只考虑拼在前面的那个串经过内部匹配后的哈希值。问题转化为，求一些子串从左向右经过括号匹配后的哈希值，要求匹配剩下的只有左括号。
　　有一个显然的性质，每个左括号和右括号的匹配是一定的，无论你怎么选子串。
　　我们从小到大枚举子串的右端点 rrr ，同时用栈维护子串 [1,r][1,r][1,r] 匹配后的形态，对于子串 [l,r][l,r][l,r] 匹配后的形态，它只是 [l,r][l,r][l,r] 匹配后的形态的一个后缀。用二分找到具体是哪个后缀。同时，我们需要判断当前每个子串是否合法，发现只需要求出每个右括号匹配的左括号的位置的最小值，然后看它是否小于 lll，这个部分用线段树即可。

G GCD

【题意】
　　给出正整数 a≤5000,b≤1018a\le 5000,b \le 10^{18}a≤5000,b≤1018。有两种操作，分别为 a→a−gcd⁡(a,b)a\rightarrow a - \gcd(a,b)a→a−gcd(a,b) 和 b→b−gcd⁡(a,b)b\rightarrow b - \gcd(a,b)b→b−gcd(a,b) 求最少操作次数使得 a,ba,ba,b 均变为 000。
【题解】
　　可以视为 000 的代价使 a,ba,ba,b 除以 gcdgcdgcd，111 的代价使任一减一。然后我们注意到在这样的过程中，至多两次操作就可以使 a,ba,ba,b 的 gcd⁡≥2\gcd \ge 2gcd≥2，而 a≤5000a \le 5000a≤5000 所以至多只需 242424 次操作即可使 aaa 变为除以 gcd⁡\gcdgcd 意义下的 111，此后再两次操作即可。那么答案不会超过 262626，爆搜即可。
　　赛时的想法是，gcd⁡\gcdgcd 只会变大，考虑 a,ba,ba,b 在 gcd⁡\gcdgcd 较小时的所有操作，一定是只会为了使 gcd⁡\gcdgcd 变大，而不会仅仅为了减小数值，这样更优。那么考虑 a,ba,ba,b 从 gcd⁡\gcdgcd 小的走到大的，由于 a,ba,ba,b 总是 gcd⁡\gcdgcd 的倍数，所以认为任意路径，gcd⁡\gcdgcd 一定时 a,ba,ba,b 也一定。 那么只需枚举约数转移。
　　错误的原因在于转移的过程，例如我们需要让 aaa 减 333 次，bbb 减 222 次转移到另一个 gcd⁡\gcdgcd，但减的顺序会有关，可能存在路径使得减了一半 gcd⁡\gcdgcd 就变为另一个数，但也可能存在需要的路径。所以转移过程中需要进行判断是否存在时时互质的路径。 这个判断并不容易做。

J. 四舍五入

【题目大意】：

对于一个整数 x ，你可以进行以下操作任意次：
选择一个不超过 m 的进制 k ，将 x 写作 k 进制的形式，然后“四舍五入”使得 x 的最低位为 0 。
给定 m 和 q 组询问，每组询问想知道 x 变为 y 所需的最小操作次数。

F. 腐蚀与膨胀

【题目大意】

​ 有一个 n×nn\times nn×n 的 0/10/10/1 图，每次 0/10/10/1 以 kkk 的切比雪夫距离扩张，求 qqq 次操作后图的形态。

​ n≤500,q≤1e6n\le 500,q \le 1e6n≤500,q≤1e6

【题解】

​ 小智慧题。赛场上过的队伍很少。

​ 单次扩张操作可以利用二维前缀和做到 n2n^2n2。

​ 用 (0/1,k)(0/1,k)(0/1,k) 表示每次操作。

​ 发现连续两个操作类型一样的可以合并，(0/1,a)+(0/1,b)=(0/1,a+b)(0/1,a)+(0/1,b) = (0/1,a + b)(0/1,a)+(0/1,b)=(0/1,a+b)。

​ 另外，若有 (0,a)+(1,c)+(0,b)(0,a)+(1,c) + (0,b)(0,a)+(1,c)+(0,b) 且 a≥c≤ba \ge c \le ba≥c≤b，那么 (0,a)+(1,c)+(0,b)=(0,a+b−c)(0,a)+(1,c)+(0,b) = (0,a + b - c)(0,a)+(1,c)+(0,b)=(0,a+b−c)

​ 利用上诉两个结论将操作合并，那么最后操作一定是类型 0/10/10/1 交替，且 kkk 先从小到大，在从大到小，最多 2n2n2n 个。

H. Heavy - light Decomposition

​ 将 nnn 个点分段，每段连续，视为一条链。

​ 求是否可以将这些链视为轻重链剖分中的重链并拼成一棵树。

​ 我们考虑每个点的深度越深显然重链这一约束会越大，那么考虑拿最长的链的一端作为根节点。此时发现如果根节点一定且这条链最长，那么假如不再改变这条链那么最大的子树就确定了。也就是说我们考虑把所有链拼成菊花。

​ 那么，如果最长的链（记长度为 KKK）只有一条，则其他所有链拼到根节点上即可。

​ 然后考虑如果有多余一条长为 KKK 的链，我们尝试在第一条的第二个点即子树的根上拼一条小链，然后就能保证重链最重，此时要求存在一条长度小于 K−1K - 1K−1 的链，如果不存在，那么 IMPOSSIBLE，否则拼上之后再把剩余的拼成菊花即可。
